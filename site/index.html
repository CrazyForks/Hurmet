This week’s example:

Shared Variables

Say that you are working on a large project. For this project, you and your team will prepare several calculation packages and there a several variables that ought to contain the same values in each document.

With Hurmet, you can define those values in one document. In my own engineering practice, we call them “parent files”, because the other documents inherit the parent’s values. Then each of your calculation documents can import the shared values in two lines of code.

Here’s how it can work.

First, create a text file that sets your values. I’ve put an example of such a file, containing some structural engineering values, at https://gist.githubusercontent.com/ronkok/cbbf6cde15ac1b4c1e65cc338970043a/raw/a8b6bc59a7ee0176c34d646ad539a555a5816ee3/ParentTrial.hrms.

Then, in each calculation document, write two lines of code in Hurmet calculation zones. The first line fetches the parent values. In our example, that line would be written as:

projectParameters = fetch("https://gist.githubusercontent.com/ronkok/cbbf6cde15ac1b4c1e65cc338970043a/raw/1701022ed4d571b82fd0afcac182e0b430fa17c5/ParentTrial.hrms") = !

Note the “= !” at the end of the statement. That will force an evaluation and suppress the value display.

Next, we write a line that assigns values to individual variables. An individual calculation document might use only some of the parent values. A document concerned with concrete design might be written like this:

f_c′, f_c′′, f_yr, ρ_0, ρ_max, E_c, n_c = projectParameters["f_c′", "f_c′′", "f_yr", "ρ_0", "ρ_max", "E_c", "n_c"] = !

Hurmet will render those two lines of code this way:

Now the variables can be called:

f_c′ = ? psi

E_c = ? ksi
